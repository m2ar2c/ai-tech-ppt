CPU 与 GPU 架构详细对比

Detailed Architecture Comparison

CPU：控制中心

CPU 是计算机的"控制中心"，其设计追求通用性和灵活性。CPU 拥有少量但强大的计算核心（通常为 4-16 个），每个核心具有较高的主频和复杂的流水线控制，能够快速完成依赖性强的计算任务。

核心特点

  • • 4-16 个强大核心
  • • 高主频
  • • 复杂流水线控制
  • • 大容量缓存（L1/L2/L3）

性能优势

  • • 低延迟
  • • 强单线程性能
  • • 复杂逻辑判断
  • • 灵活任务调度

GPU：并行引擎

相比之下，GPU 的设计目标是大规模并行计算。GPU 拥有大量相对简单的计算单元（通常为数千个流处理器），采用长延时流水线以实现高吞吐量。GPU 将 80% 以上的芯片面积用于 ALU（算术逻辑单元），而 CPU 则将更多面积用于缓存和控制单元。

核心特点

  • • 数千个流处理器
  • • 长延时流水线
  • • 80%+ 面积用于 ALU
  • • 简化的控制逻辑

性能优势

  • • 极高吞吐量
  • • 强大计算密度
  • • 优秀能效比
  • • 适合数据并行

应用场景对比

CPU 适用场景

复杂逻辑判断、任务调度、多任务处理、操作系统运行、数据库事务处理

GPU 适用场景

矩阵运算、图像处理、深度学习训练与推理、科学计算、视频编解码

GPU 并行计算机制详解

In-depth Parallel Computing Mechanisms

单指令多数据（SIMD）架构

SIMD 是 GPU 并行计算的基础。在这种架构中，多个计算单元同时执行相同的指令，但处理不同的数据。这与 CPU 的复杂指令集形成鲜明对比。

工作原理：

当 GPU 执行一条指令（如"将两个数相加"）时，成百上千个计算单元会同时执行这条指令，但每个单元处理不同的数据对。这种方式特别适合处理大规模数据并行任务，如矩阵乘法、图像卷积等。

流式多处理器（SM）

SM 是 NVIDIA GPU 架构的核心。每个 SM 都是一个高度并行的处理单元，包含多个 CUDA 核心、共享内存、寄存器等组件。

Ampere 架构 SM 组成：

64 个 FP32 核心

单精度浮点运算，用于深度学习训练的主要计算

64 个 INT32 核心

整数运算，用于地址计算和索引操作

32 个 FP64 核心

双精度浮点运算，用于科学计算高精度需求

4 个 Tensor Core

专用张量计算单元，大幅加速矩阵乘法

线程层次结构

CUDA 采用 Grid-Block-Thread 的三级结构。线程被组织成线程块（Thread Block），多个线程块构成一个网格（Grid）。

Warp（线程束）

每个 CUDA 核心可以并发执行多个线程，通常以 32 个线程为一组，称为 Warp。Warp 是 GPU 调度和执行的基本单位。

Thread Block（线程块）

多个线程组成一个线程块，块内线程可以通过共享内存协作。线程块被分配到 SM 上执行。

Grid（网格）

多个线程块构成网格，网格定义了整个并行计算任务的规模。

GPU 并行计算三大优势

极高的吞吐量

能够同时处理大量数据，适合大规模并行任务

强大的计算密度

通过大量计算单元实现惊人的峰值算力

优秀的能效比

在处理并行任务时，单位功耗下的计算能力远超 CPU

CUDA 编程模型详解

CUDA Programming Model Deep Dive

CUDA 简介

CUDA（Compute Unified Device Architecture）是 NVIDIA 推出的并行计算平台和编程模型，它让开发者能够使用 C/C++、Python 等熟悉的编程语言直接控制 GPU 进行通用计算。

SM 结构基础

CUDA 的底层实现基于 SM（流式多处理器）结构。每个 SM 包含多个 CUDA 核心，这些核心能够并行执行大量线程。

SM 组件：

  • • CUDA 核心：基本计算单元，执行浮点和整数运算
  • • 共享内存：SM 内线程共享的高速缓存，延迟低于全局内存
  • • 寄存器：每个线程的私有存储空间，速度最快
  • • L1 缓存：一级缓存，加速数据访问
  • • 线程调度器：管理 Warp 调度和指令分发

PTX 中间表示

CUDA 程序最终被编译成 PTX（Parallel Thread Execution）中间表示，这是一种用于 CUDA 设备代码的虚拟指令集架构。

PTX 的优势：

  • 硬件独立性：PTX 代码可以在不同代的 NVIDIA GPU 上运行，驱动会在运行时将 PTX 编译为特定硬件的机器码
  • 前向兼容：为旧架构编译的 PTX 代码可以在新架构上运行并获得优化
  • 优化机会：驱动可以针对具体硬件进行运行时优化

线程调度机制

线程调度是 CUDA 高效运行的关键。每个 SM 有 两个线程束调度器和两个指令调度单元。

调度流程：

1. 线程块分配

当线程块被分配给 SM 时，线程块内的所有线程被分成多个 Warp（每 Warp 32 个线程）

2. Warp 选择

调度器从准备就绪的 Warp 中选择一个执行指令

3. 隐藏延迟

当一个 Warp 因等待数据而暂停时，调度器立即切换到另一个准备就绪的 Warp，保持计算单元始终处于工作状态

4. 指令执行

Warp 内的 32 个线程同时执行相同指令（SIMD 模式）

延迟隐藏技术

通过维护大量并发线程（Warp），当一个 Warp 等待内存访问时，调度器可以立即切换到其他就绪 Warp 继续计算。这种设计使得 GPU 即使在较高的内存访问延迟下，也能维持很高的计算效率。这是 GPU 能够高效处理大规模并行任务的核心技术之一。

HBM 高带宽内存技术详解

High Bandwidth Memory Technology Deep Dive

HBM 技术概述

HBM（高带宽内存）代表了内存技术的最新发展。HBM 采用 3D 堆叠架构和 TSV（硅通孔）技术，通过垂直堆叠多层 DRAM 芯片并使用硅通孔实现层间互连，大幅缩短了数据传输路径。

HBM 核心技术

3D 堆叠架构

将多层 DRAM 芯片垂直堆叠在一起，每层之间通过 TSV（Through-Silicon Via，硅通孔）连接。这种堆叠方式大幅减少了芯片占用面积，同时显著缩短了数据传输距离。

TSV 硅通孔技术

在硅晶圆中打通垂直孔洞并填充导电材料，实现芯片层间的高速互连。TSV 技术相比传统的引线键合，具有更短的信号路径、更低的功耗和更高的带宽。

宽位宽接口

HBM 使用 1024 位或更宽的数据接口（相比 GDDR6 的 32 位），虽然频率相对较低，但通过超宽位宽实现了极高的聚合带宽。

HBM 发展历程

HBM2

2016

首次商用版本，提供 256GB/s 带宽，容量 8GB。相比 GDDR5 实现了显著的带宽提升和功耗降低。

HBM2e

2018

增强版本，传输速度提升至 3.6Gbps，容量扩展到 16GB，带宽达到 461GB/s。开始大规模应用于 AI 训练。

HBM3

2022

第三代产品，采用改进的 2.5D/3D 架构，带宽突破 600GB/s，容量可达 24GB。能效比再次提升。

HBM3e

Latest

最新版本，传输速率达到 8Gbps，最高每秒可处理 1.15-1.225TB 数据。成为大规模 AI 训练的标配，显著降低了训练时间和能耗。

HBM 对 AI 的重要性

内存带宽是影响 GPU 性能的关键瓶颈之一。在大模型训练中，模型参数和激活值的频繁读写对内存带宽提出了极高要求。HBM 的超高带宽和低延迟特性使得 GPU 能够更快地获取数据、完成计算，从而显著缩短训练时间、降低能耗，并支持更大规模的模型训练。HBM 已经成为现代 AI 算力卡的标配技术。

NVLink 技术演进与应用

NVLink Technology Evolution and Applications

NVLink 技术概述

NVLink 是英伟达开发的高速 GPU 互联技术，采用高速差分信号传输，支持多链路聚合。NVLink 实现了 GPU 间的高速低延迟通信，构建了统一的内存空间。

核心技术特点

高速差分信号传输

NVLink 采用高速差分信号技术，每条链路由多对差分信号线组成。差分信号具有更强的抗干扰能力和更低的电磁辐射，能够支持更高的传输速率。

多链路聚合

两个 GPU 之间可以部署多条 NVLink 链路并行工作，实现带宽聚合。例如，8 条链路可以提供 8 倍于单链路的带宽，同时增强了容错能力。

统一内存空间

通过 NVLink 连接的多个 GPU 可以形成统一的内存地址空间，GPU 可以直接访问其他 GPU 的显存，无需通过 CPU 中转，大幅降低了通信延迟和开销。

点对点直连

GPU 之间通过 NVLink 直接连接，无需经过 PCIe 总线和 CPU。这种点对点拓扑大幅降低了通信路径上的延迟和带宽瓶颈。

NVLink 代际演进

Gen 1

Pascal 架构，单链路 20GB/s，主要用于 P100

Gen 2

Volta 架构，单链路 25GB/s，应用于 V100，支持 6 条链路

Gen 3

Ampere 架构，单链路 50GB/s，应用于 A100，支持 12 条链路，总带宽 600GB/s

Gen 4

Hopper 架构，单链路 100GB/s，应用于 H100，总带宽可达 900GB/s

Gen 5

最新一代，提供 1.8TB/s 的双向带宽，可扩展至 576 个 GPU 集群，为超大规模 AI 训练提供基础设施支持

NVLink vs PCIe

PCIe 4.0 x16

  • • 带宽：~32GB/s（双向）
  • • 需经过 CPU
  • • 延迟较高
  • • 通用标准

NVLink Gen 5

  • • 带宽：1.8TB/s（双向）
  • • GPU 直连
  • • 延迟极低
  • • 专用技术

NVLink 对大规模 AI 训练的意义

在大规模分布式训练中，GPU 之间需要频繁交换梯度、参数等数据。传统的 PCIe 总线带宽有限且需要经过 CPU 中转，成为严重的性能瓶颈。

NVLink 的超高带宽和低延迟特性使得 GPU 集群能够高效协作，大幅缩短训练时间。例如，在训练 GPT-3 这样的千亿参数模型时，NVLink 的通信效率直接决定了训练的可行性和成本。第五代 NVLink 支持的 576 GPU 集群规模，为未来更大规模模型的训练奠定了硬件基础。
